你描述的现象（monorepo + pnpm workspace 里跑报错，但把项目拷贝到独立目录就正常），即使你只有一个 Vue3 项目、vue-router 也是 4.3.*，最常见根因也不是“版本错”，而是：

拷贝到独立目录后，node_modules 变成了 npm/yarn 那种“扁平 hoist 结构”；而 pnpm workspace 默认是严格的 symlink + 非扁平结构。你的构建/运行工具（你说不是 Vite）很可能没完全适配 pnpm 的 node_modules 结构，所以解析到的入口文件/依赖路径不同，从而触发 vue-router 相关报错。

下面给你几个“命中率最高、改动最小”的修复方案（从最省事到最彻底）。

⸻

方案 1：让 pnpm 变成 “hoist 模式”（最像你拷贝后能正常的环境）

在 monorepo 根目录创建/修改 .npmrc：

node-linker=hoisted

然后在根目录重新安装：

rm -rf node_modules pnpm-lock.yaml
pnpm install

再启动你的子项目（一定用 -C 或 -F）：

pnpm -C packages/你的项目名 dev
# 或
pnpm -F 你的包名 dev

这一步经常“一键治好” webpack / vue-cli / rollup 老配置下的各种 router/runtime 报错，因为它把 node_modules 结构变得更像 npm/yarn。

⸻

方案 2：如果你用的是 Vue CLI（Webpack），加两处配置（适配 pnpm symlink）

在 packages/你的项目/vue.config.js 加：

const path = require("path");

module.exports = {
  // ✅ 关键1：让 webpack 解析 symlink 时不要“绕出”真实路径导致多份依赖/注入异常
  configureWebpack: {
    resolve: {
      symlinks: false,
      alias: {
        // ✅ 关键2：强制 vue / vue-router 只走同一份（workspace 根 node_modules）
        vue: path.resolve(__dirname, "../../node_modules/vue"),
        "vue-router": path.resolve(__dirname, "../../node_modules/vue-router"),
      },
    },
  },
};

然后重新启动。

symlinks: false + alias 这套，在 pnpm workspace 下对 “router 注入拿不到 / useRouter undefined / currentRoute undefined / 奇怪运行时错误” 特别有效。

⸻

方案 3：你不是 Vite，可能是 Webpack “入口字段”解析差异，直接把 vue-router 指到稳定入口

有些构建环境会在 pnpm 结构下解析到 .mjs 或某个 bundler 入口，结果不兼容。你可以硬指定 vue-router 的 bundler 入口：

webpack resolve alias 增强：

alias: {
  "vue-router$": path.resolve(__dirname, "../../node_modules/vue-router/dist/vue-router.esm-bundler.js"),
}

这招适用于：报错看起来像 export 语法解析失败、或模块入口解析不一致导致的异常。

⸻

方案 4：确认你确实是在“子项目目录”启动（否则很多工具会把 root 当工程根）

在 monorepo 根运行 pnpm dev 很容易把 root 当成 cwd，导致解析错位。

务必这样启动：

pnpm -C packages/你的项目名 dev

或根 package.json 写死脚本：

{
  "scripts": {
    "dev": "pnpm -C packages/你的项目名 dev"
  }
}


⸻

你可以用 2 条命令快速定位是哪一类问题（不需要贴长日志）

在根目录跑：

pnpm -C packages/你的项目名 why vue
pnpm -C packages/你的项目名 why vue-router

	•	如果输出里出现多条不同路径来源（哪怕版本相同），也可能是 “多份实例/不同解析路径” 引起的注入异常 → 优先用 方案 1 或 2。

⸻

我给你一个最推荐的执行顺序（基本不费脑）
	1.	先上 方案 1（.npmrc node-linker=hoisted + 重装）
	2.	还不行就上 方案 2（vue.config.js 的 symlinks:false + alias）
	3.	再不行就上 方案 3（硬指 vue-router bundler 入口）
