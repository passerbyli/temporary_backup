```
cd packages\vue3-web1
node -p "require.resolve('vue')"
node -p "require.resolve('@vue/runtime-core')"
```
检查两个项目路径不同
但接下来要验证：web1 打包时是不是用的这条路径。如果不是，说明 webpack/cwd/context 串了。

在 packages/vue3-web1/webpack.config.js 顶部加：
```
console.log('[web1 webpack] cwd=', process.cwd())
console.log('[web1 webpack] vue=', require.resolve('vue'))
console.log('[web1 webpack] runtime-core=', require.resolve('@vue/runtime-core'))
```
如果这里打印出来的路径不是 Step1 在 web1 目录下的那条，就坐实：
webpack 解析串包 / cwd 不在 web1

这也是你“加 web2 才坏”的最常见原因：加入 web2 后，workspace 根依赖树变化，webpack 从根解析到了 SDK 带的 vue。

Step 4：检查 pnpm 依赖树是否出现多份 Vue runtime（辅助证据）
```
pnpm -r list vue
pnpm -r list @vue/runtime-core
pnpm why vue
pnpm why @vue/runtime-core
```
